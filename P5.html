P5 - Procesos M/M/s¶

Es menester tomar una decisin con respecto al nmero de "servidores" necesarios para cierto proceso de vacunacin, de forma tal que el sistema no exceda 50 personas en fila durante el 95% del tiempo de servicio.

En un estudio de afluencia y del tiempo de servicio se asumen y determinan los siguientes parmetros:

  • La tasa de llegada $\lambda$ es constante durante ocho horas de funcionamiento y tiene un valor de $\lambda = 4$ personas por minuto.
  • La tasa de servicio $\nu$ de cada "servidor" (es decir, personal de salud tomando datos y vacunando) es constante durante ocho horas de funcionamiento y tiene un valor de $\nu = 0.25$ personas por minuto.

(40%) Determinación teórica del número $s$ de servidores necesarios para cumplir el requisito.¶

Se tienen los siguientes datos del enunciado anterior:

  • El sistema no debe exceder un total de 50 personas en fila durante el 95% del tiempo de servicio.
  • La tasa de llegada tiene un valor de $\lambda = 4$ personas por minuto durante las ocho horas de funcionamiento.
  • La tasa de servicio de cada servidor tiene un valor de $\nu = 0.25$ personas por minuto durante las ocho horas de funcionamiento.

La longitud de la fila $L_q$ viene dada por la siguiente expresin:

$$ L_q = i - 1 $$

Tomando en cuenta una cantidad mxima de 50 personas en fila, entonces el nmero de pacientes en el sistema ($i$) corresponde a:

$$ i = L_q + 1 $$$$ i = 50 + 1 = 51 $$

La probabilidad de que haya 51 o ms pacientes en el sistema viene dada por la siguiente expresin:

$$ P( \text{51 o pacientes en el sistema} ) = \sum_{i=51}^{\infty} (1 - \rho) \rho^i = 1 - \sum_{i=0}^{50} (1 - \rho) \rho^i = \rho^ {51} $$

Donde el parmetro $\rho$ est definido de la siguiente forma, donde s representa la cantidad de servidores disponibles.

\begin{equation} \rho=\frac{\lambda}{s\cdot \nu} \end{equation}

Sustituyendo esto en la expresin anterior:

$$ P( \text{51 o ms pacientes en el sistema} ) = \rho^ {51} = \left( \frac{\lambda}{s \nu} \right)^{51} $$

Se busca no sobrepasar un total de 50 personas en espera durante el 95% del tiempo; es decir, tener ms de 50 pacientes en espera solo el 5% del tiempo. De esta forma, se puede establecer la siguiente expresin:

$$ \left( \frac{\lambda}{s \nu} \right)^{51} \leq 0.05 $$

Despejando la cantidad de servidores $s$:

$$ \frac{ \lambda^{51} }{ s^{51} \nu ^{51} } \leq 0.05 $$$$ \frac{ \lambda^{51} }{ 0.05 \cdot \nu ^{51} } \leq s^{51} $$$$ s^{51} \geq \left(\frac{ \lambda }{ \nu } \right)^{51} \cdot \left(\frac{1}{0.05}\right) $$$$ s \geq \left(\frac{ \lambda }{ \nu } \right) \cdot \left(\frac{1}{0.05}\right)^{\frac{1}{51}} $$

De los datos del enunciado, se tiene que la tasa de llegada es $\lambda = 4$ y la tasa de servicio es $\nu = 0.25$.

$$ s \geq \left(\frac{4}{0.25} \right) \cdot \left(\frac{1}{0.05}\right)^{\frac{1}{51}} = 16.97 $$

En este caso, los servidores seran personas en el centro de vacunacin, por lo que solo tiene sentido hablar de nmeros enteros. De esta forma, se redondea el nmero a 17 y se concluye que esta es la cantidad mnima de servidores para mantener a 50 personas en fila durante el 95% del tiempo.

import numpy as np
from scipy import stats
import matplotlib.pyplot as plt

def vacunacion(s):

    '''Esta funcin realiza una simulacin del proceso de vacunacin
    Recibe como entrada en nmero de servidores s y grafica la
    cantidad de pacientes en el sistema a lo largo del tiempo'''

    # Nmero de pacientes                        ########## Cul debera ser el nmero de pacientes? en ningn lado lo dicen
    N = 1900

    # Parmetro de llegada (pacientes/segundos)
    lam = 4/60

    # Parmetro de servicio (servicios/segundos)
    nu = (0.25*s)/60

    # Distribucin de los tiempos de llegada entre cada paciente
    X = stats.expon(scale = 1/lam)

    # Distribucin de los tiempos de servicio a cada paciente
    Y = stats.expon(scale = 1/nu)

    # Intervalos entre llegadas (segundos desde ltimo paciente)
    t_intervalos = np.ceil(X.rvs(N)).astype('int')

    # Tiempos de las llegadas (segundos desde el inicio)
    t_llegadas = [t_intervalos[0]]
    for i in range(1, len(t_intervalos)):
        siguiente = t_llegadas[i-1] + t_intervalos[i]
        t_llegadas.append(siguiente)

    # Tiempos de servicio (segundos desde inicio de servicio)
    t_servicio = np.ceil(Y.rvs(N)).astype('int')

    # Inicializacin del tiempo de inicio y fin de atencin
    inicio = t_llegadas[0]          # primera llegada
    fin = inicio + t_servicio[0]    # primera salida

    # Tiempos en que recibe atencin cada i-simo paciente (!= que llega)
    t_atencion = [inicio]
    for i in range(1, N):
        inicio = np.max((t_llegadas[i], fin))
        fin = inicio + t_servicio[i]
        t_atencion.append(inicio)

    # Inicializacin del vector temporal para registrar eventos
    t = np.zeros(t_atencion[-1] + t_servicio[-1] + 1)

    # Asignacin de eventos de llegada (+1) y salida (-1) de pacientes
    for c in range(N):
        i = t_llegadas[c]
        t[i] += 1
        j = t_atencion[c] + t_servicio[c]
        t[j] -= 1

    # Umbral de P o ms personas en sistema (hay P - 1 en fila)
    P = 51

    # Instantes (segundos) de tiempo con P o ms solicitudes en sistema
    exceso = 0

    # Proceso aleatorio (estados n = {0, 1, 2...})
    Xt = np.zeros(t.shape)

    # Inicializacin de estado n
    n = 0

    # Recorrido del vector temporal y conteo de clientes (estado n)
    for i, c in enumerate(t):
        n += c # sumar (+1) o restar (-1) al estado
        Xt[i] = n
        if Xt[i] >= P: 
            exceso += 1

    # Fraccin de tiempo con P o ms solicitudes en sistema
    fraccion = exceso / len(t)

    # Resultados
    print('Parmetro lambda =', str(lam*60))
    print('Parmetro nu =', str(nu*60))
    print('Nmero de servidores =', str(s)) 
    print('Tiempo con ms de {} solicitudes en fila:'.format(P-1))      ###### Ac correg esto a P-1 en vez de P-2
    print('\t {:0.2f}%'.format(100*fraccion))
    if fraccion <= 0.05:
        print('\t S cumple con la especificacin.')
    else:
        print('\t No cumple con la especificacin.') 
    print('Simulacin es equivalente a {:0.2f} horas.'.format(len(t)/3600))
    
    Title = " Simulacin de vacunacin para " + str(s) + " servidores."
    # Grfica de X(t) (estados del sistema)
    plt.figure()
    plt.plot(Xt)
    plt.plot(range(len(t)), (P-1)*np.ones(t.shape))
    plt.legend(('$X(t) = n$', '$L_q = $' + str(P-1)))     ###### Ac correg esto a P-1 en vez de P-2
    plt.ylabel('Pacientes en el sistema, $n$')
    plt.xlabel('Tiempo, $t$ / segundos')
    plt.title(Title)
    plt.xlim((0, 28800)) ################## Ac debera ser 8 horas en segundos?? es que si pongo 28800s se sale del rango
    plt.show()

Análisis de los resultados¶

Observando la ejecucin de la funcin con 16 servidores se observa que el nmero de pacientes excede 50 en diversos instantes de tiempo. Si nos apegamos al eneunciado, estos instantes de tiempo pueden corresponder al 5% del tiempo que se permite para que hayan ms de 50 pacientes por lo que se podra afirmar que este es un resutado aceptable. Sin embargo, al ejecutar la simulacin en repetidas ocasiones podemos ogbservar que no se puede garantizar que el 100% de las veces se cumplir con el requesito solicitado, entonces se pueden descartar los 16 servidores. Por otro lado, se observa que para 17 servidores el nmero de pacientes presentes en todo momento es menor a 50, por lo que es posible concluir que el valor calculado para los servidores fue correcto y en este caso tenemos un funcionamiento ideal.

Agregar ms servidores disminuye el nmero de pacientes de manera general, pero esto requiere una mayor inversin y por lo tanto no es recomendable.
